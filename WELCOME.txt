🔌 API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

POST /api/bpm
  ← ESP32 sends BPM data here
  Request: {"bpm": 78, "raw": 1023, "baseline": 1020, "device": "R0384-ESP32C3"}
  Response: {"success": true, "message": "BPM recorded"}

GET /api/latest
  → Get current BPM reading
  Response: {"bpm": 78, "raw": 1023, "baseline": 1020, "device": "...", "timestamp": "..."}

POST /api/session
  ← Therapist updates VR config
  Request: {"speech_script": "...", "crowd_count": 5, "volume_level": 0.7}
  Response: {"success": true, "data": {...}}

GET /v1/session/{sessionId}
  → Unity polls this every 1-2 seconds
  Response: {"sessionId": "default", "speech_script": "...", "crowd_count": 5, ...}

GET /api/stats
  → Dashboard analytics
  Response: {"current_bpm": 78, "session_config": {...}, "server_uptime": ...}

GET /health
  → Health check
  Response: {"status": "ok", "timestamp": "..."}

WebSockets:
  bpm_update - Real-time BPM broadcasts
  session_update - Real-time config changes

📱 INTEGRATION PATHS
═══════════════════════════════════════════════════════════════════════════════

Path 1: Using ESP32 Wearable
─────────────────────────────
  1. Upload ESP32_EXAMPLE.ino to your ESP32 board (Arduino IDE)
  2. Configure WiFi SSID/password in the code
  3. Configure backend URL (your computer's IP or domain)
  4. Upload & restart ESP32
  5. Check backend console for "Received BPM: X"
  6. Watch dashboard update in real-time

Path 2: Using Unity VR
──────────────────────
  1. Create new C# script in Unity with code from UNITY_INTEGRATION.cs
  2. Attach SessionManager script to an empty GameObject
  3. Drag references to your crowd manager, audio manager, text display
  4. Hit Play
  5. Unity will poll backend every 1-2 seconds
  6. Scene updates based on therapist's controls

Path 3: Testing Without Hardware
────────────────────────────────
  1. Use curl commands to send BPM data
  2. Watch dashboard update in real-time
  3. Adjust sliders on dashboard
  4. Test all endpoints
